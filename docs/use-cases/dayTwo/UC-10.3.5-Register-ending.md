# UC-10.3.5 — Registrierung abschliessen (Confirm-Endpoint)

## Ziel

Der Benutzer klickt auf den Bestätigungslink in der E-Mail. Der Confirm-Endpoint **validiert den JWT-Token kryptografisch**, extrahiert die gespeicherten Registrierungsdaten aus den Claims, legt den Account dauerhaft in der Datenbank an und gibt ein Auth-JWT zurück, mit dem der Benutzer sofort eingeloggt ist.

---

## Geplanter Workflow (Schritt für Schritt)

```
GET /auth/register/confirm/{token}   (token = Registrierungs-JWT)
         |
         v
┌─────────────────────────────────────────────┐
│  AuthController.confirm()                   │
│                                             │
│  ↓ ruft auf: RegistrierungsService          │
│                                             │
│  5. Bestätigung oder Fehler zurückgeben     │
│     → 201 Created + Auth-JWT  (Erfolg)      │
│     → 404 Not Found  (Signatur ungültig /   │
│                        bereits aktiviert)   │
│     → 410 Gone       (Token abgelaufen)     │
└─────────────────────────────────────────────┘
         |
         v
┌─────────────────────────────────────────────┐
│  RegistrierungsService.bestaetigen()        │
│                                             │
│  1. JWT parsen + Signatur prüfen            │
│     → JwtException wenn ungültig/gefälscht  │
│                                             │
│  2. exp-Claim prüfen (automatisch via JWT)  │
│     → ExpiredJwtException wenn abgelaufen   │
│                                             │
│  3. type-Claim prüfen                       │
│     → muss "registration" sein              │
│                                             │
│  4. Username-Eindeutigkeit prüfen           │
│     → falls bereits in DB: schon aktiviert  │
│                                             │
│  5. PendingRegistration aus Claims bauen    │
│     (username, email, pwHash)               │
└─────────────────────────────────────────────┘
         |
         v
┌─────────────────────────────────────────────┐
│  AccountService.erstelleAccount()           │
│                                             │
│  6. Account dauerhaft in DB speichern       │
└─────────────────────────────────────────────┘
```

---

## Ist-Zustand vs. Soll-Zustand

| Schritt | Ist-Zustand (aktuell) | Soll-Zustand |
|---|---|---|
| Token-Typ | `UUID` (Path-Variable) | `String` JWT (Path-Variable) |
| Token-Validierung | Map-Lookup in pending | JWT-Signatur + `exp` via `JWTUtil` |
| Ablaufprüfung | manueller `Instant`-Vergleich | `ExpiredJwtException` automatisch |
| Token unbekannt/gefälscht | `TokenNotFoundException` → 404 | `JwtException` → 404 |
| Token abgelaufen | `TokenExpiredException` → 410 | `ExpiredJwtException` → 410 |
| Duplicate-Check | beim Register in pending-Map | beim Confirm in DB |
| Account anlegen | direkt im Controller | `AccountService.erstelleAccount()` |
| Map aufräumen | `pending.remove(token)` | **entfällt** (kein Map-Zustand) |

## Was sich ändern muss

### `RegistrierungsService` — `bestaetigen(String token)`

```java
public PendingRegistration bestaetigen(String token) {
    Claims claims;
    try {
        claims = jwtUtil.parseRegistrationToken(token);
        // ExpiredJwtException wird separat behandelt (→ 410)
    } catch (ExpiredJwtException e) {
        throw new TokenExpiredException();
    } catch (JwtException e) {
        throw new TokenNotFoundException();
    }

    String username = claims.getSubject();
    String email    = claims.get("email",  String.class);
    String pwHash   = claims.get("pwHash", String.class);

    if (accountRepository.existsByUsername(username)) {
        throw new UsernameAlreadyExistsException(username);   // Link bereits verwendet
    }

    return new PendingRegistration(username, email, pwHash);
}
```

### `AccountService` — Account anlegen

```java
@Service
public class AccountService {
    private final AccountRepository accountRepository;

    public Account erstelleAccount(PendingRegistration pending) {
        Account account = new Account();
        account.setUsername(pending.getUsername());
        account.setPasswordHash(pending.getPasswordHash());
        account.setEmail(pending.getEmail());
        return accountRepository.save(account);
    }
}
```

### `AuthController` — Confirm-Endpoint

```java
@GetMapping("/register/confirm/{token}")
public ResponseEntity<?> confirm(@PathVariable String token) {
    try {
        PendingRegistration pending = registrierungsService.bestaetigen(token);
        Account account = accountService.erstelleAccount(pending);
        TokenData jwt = new TokenData(jwtUtil.generateToken(account.getUsername()));
        return ResponseEntity.status(HttpStatus.CREATED).body(jwt);
    } catch (TokenNotFoundException | UsernameAlreadyExistsException e) {
        return ResponseEntity.status(HttpStatus.NOT_FOUND).build();
    } catch (TokenExpiredException e) {
        return ResponseEntity.status(HttpStatus.GONE).build();
    }
}
```

## HTTP-Statuscodes

| Situation | Code | Bedeutung |
|---|---|---|
| Account erfolgreich angelegt | `201 Created` | Benutzer ist direkt eingeloggt (JWT im Body) |
| Token ungültig / bereits aktiviert | `404 Not Found` | Signatur fehlerhaft, gefälscht oder Link schon genutzt |
| Token älter als 24h | `410 Gone` | Link abgelaufen, neue Registrierung nötig |
