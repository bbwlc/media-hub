# UC-10.3.4 — Register-Workflow: Aufgabenteilung Controller / Service

## Ziel

Der `POST /auth/register`-Endpoint soll nach dem **Single-Responsibility-Prinzip** umgebaut werden: Der Controller übernimmt nur noch Validierung und HTTP-Antwort, die gesamte Business-Logik wandert in den `RegistrierungsService`. Gleichzeitig wird der UUID-basierte Pending-Token durch einen **signierten JWT** ersetzt — kein serverseitiger Map-Zustand mehr.

---

## Geplanter Workflow (Schritt für Schritt)

```
POST /auth/register  { username, password, email }
         |
         v
┌─────────────────────────────────────────────┐
│  AuthController.register()                  │
│                                             │
│  1. Pflichtfelder & E-Mail-Format prüfen    │
│     → 400 Bad Request wenn ungültig         │
│                                             │
│  ↓ ruft auf: RegistrierungsService          │
│                                             │
│  4. 202 Accepted zurückgeben                │
│     (noch KEIN Account in der DB!)          │
└─────────────────────────────────────────────┘
         |
         v
┌─────────────────────────────────────────────┐
│  RegistrierungsService.starteRegistrierung()│
│                                             │
│  2. Username-Eindeutigkeit prüfen           │
│     → Account-Tabelle (bereits registriert?)│
│     → Exception wenn bereits vorhanden      │
│                                             │
│  3. Passwort hashen                         │
│     JWT-Token erzeugen mit Claims:          │
│       sub=username, email, pwHash, exp=+24h │
│       type="registration"                   │
│                                             │
│  4. MailService.sendRegistrationVerification│
│         (email, jwtToken)                   │
│     → Link mit JWT in Konsole ausgeben      │
└─────────────────────────────────────────────┘
```

> **Kein Map-Zustand mehr:** Die `Map<UUID, PendingRegistration>` entfällt vollständig.
> Alle Registrierungsdaten reisen im signierten JWT vom Server zur E-Mail zum Confirm-Endpoint zurück.

---

## Ist-Zustand vs. Soll-Zustand

| Schritt | Ist-Zustand (aktuell) | Soll-Zustand |
|---|---|---|
| Pflichtfeld-Validierung | `AuthController` | `AuthController` (bleibt) |
| Username-Eindeutigkeit | `RegistrierungsService` | `RegistrierungsService` (bleibt) |
| Pending-Map | `Map<UUID, PendingRegistration>` | **entfällt** |
| Token-Typ | `UUID.randomUUID()` | `JWTUtil.generateRegistrationToken()` |
| Ablaufkontrolle | manueller `Instant`-Vergleich | `exp`-Claim im JWT |
| `MailService` aufrufen | `AuthController` | `RegistrierungsService` |
| Rückgabewert | `UUID token` | `void` (Token ist nur in der Mail) |
| HTTP-Antwort | `AuthController` | `AuthController` (bleibt) |

## Was sich ändern muss

### `JWTUtil` — neue Methoden für Registrierungs-JWT

```java
/** Erzeugt einen signierten JWT als Registrierungstoken (24h gültig). */
public String generateRegistrationToken(String username, String email,
                                        String passwordHash, long validityHours) {
    return Jwts.builder()
            .subject(username)
            .claim("email",  email)
            .claim("pwHash", passwordHash)
            .claim("type",   "registration")
            .issuedAt(new Date())
            .expiration(new Date(System.currentTimeMillis() + validityHours * 3_600_000))
            .signWith(getSignKey())
            .compact();
}

/** Parst und validiert einen Registrierungs-JWT. Wirft JwtException bei Fehler. */
public Claims parseRegistrationToken(String token) {
    Claims claims = Jwts.parser()
            .verifyWith(getSignKey())
            .build()
            .parseSignedClaims(token)
            .getPayload();

    if (!"registration".equals(claims.get("type", String.class))) {
        throw new JwtException("Not a registration token");
    }
    return claims;
}
```

### `RegistrierungsService` — kein Map-Zustand, JWT-Erzeugung

```java
@Service
public class RegistrierungsService {

    private static final long TOKEN_VALIDITY_HOURS = 24;

    private final PasswordEncoder passwordEncoder;
    private final AccountRepository accountRepository;
    private final MailService mailService;
    private final JWTUtil jwtUtil;

    public void starteRegistrierung(RegisterDto dto) {
        // Eindeutigkeit prüfen
        if (accountRepository.existsByUsername(dto.username())) {
            throw new UsernameAlreadyExistsException(dto.username());
        }

        // Passwort hashen, JWT erzeugen
        String passwordHash = passwordEncoder.encode(dto.password());
        String regToken = jwtUtil.generateRegistrationToken(
                dto.username(), dto.email(), passwordHash, TOKEN_VALIDITY_HOURS);

        // Bestätigungslink versenden (Konsole-Stub)
        mailService.sendRegistrationVerification(dto.email(), regToken);
    }
}
```

### `AuthController` — nur noch Validierung und HTTP-Antwort

```java
@PostMapping("/register")
public ResponseEntity<?> register(@RequestBody RegisterDto dto) {
    // 1. Pflichtfelder und E-Mail-Format prüfen
    if (dto.username() == null || dto.username().isBlank() ||
        dto.password() == null || dto.password().isBlank() ||
        dto.email()    == null || dto.email().isBlank()) {
        return ResponseEntity.status(HttpStatus.BAD_REQUEST).build();
    }

    try {
        registrierungsService.starteRegistrierung(dto);
    } catch (UsernameAlreadyExistsException e) {
        return ResponseEntity.status(HttpStatus.CONFLICT).build();
    }

    // 4. 202 Accepted zurückgeben (noch KEIN Account!)
    return ResponseEntity.status(HttpStatus.ACCEPTED)
            .body(Map.of("message", "Registration pending. Check your email to activate your account."));
}
```
