# UC-10.3.1 — Pending Registration (Double-Opt-In)

## Ziel

Ein neuer Benutzer registriert sich, wird aber **nicht sofort aktiviert**. Stattdessen wird ein signierter **JWT-Token** erzeugt, der alle Registrierungsdaten enthält, und ein Bestätigungslink per E-Mail verschickt. Erst nach Klick auf den Link wird der Account in der Datenbank angelegt.

---

## Warum JWT statt UUID?

| Kriterium | UUID | JWT |
|---|---|---|
| Server-seitiger Zustand | `Map<UUID, PendingRegistration>` im RAM | **keiner** — Daten im Token |
| Neustart-Sicherheit | Pending-Einträge gehen verloren | Token bleibt gültig |
| Ablaufprüfung | manuelle `createdAt + 24h`-Prüfung | `exp`-Claim, automatisch |
| Manipulationsschutz | UUID ist unratbar, aber nicht signiert | HS256-Signatur, fälschungssicher |
| URL-Länge | ~36 Zeichen | ~200+ Zeichen |
| Duplicate-Prevention | Map-Lookup beim Register | DB-Check beim Confirm |

**Designentscheid:** Wir wählen JWT für stateless Pending-Registration. Die `Map<UUID, PendingRegistration>` entfällt.

## Beteiligte Klassen

### `RegisterDto`

Eingehende Daten vom Client beim Registrierungsaufruf.

```java
public record RegisterDto(String username, String email, String password) { }
```

---

### `PendingRegistration`

Wird **nicht mehr serverseitig gespeichert** — die Felder reisen im JWT-Payload.

| JWT-Claim | Inhalt |
|---|---|
| `sub` | username |
| `email` | E-Mail-Adresse |
| `pwHash` | BCrypt-Hash des Passworts |
| `type` | `"registration"` (unterscheidet vom Auth-JWT) |
| `exp` | jetzt + 24 h |

> Hinweis: `pwHash` im JWT klingt heikel, ist es aber nicht — BCrypt ist eine Einwegfunktion.
> Der Hash reist in einer E-Mail (ggf. in Logs), aber daraus kann das Klartext-Passwort nicht
> zurückgewonnen werden. Dennoch sollte die Verbindung HTTPS sein.

---

### `RegistrierungsService`

Erzeugt den Registrierungs-JWT und validiert ihn beim Bestätigen.

```java
@Service
public class RegistrierungsService {

    private static final long TOKEN_VALIDITY_HOURS = 24;

    private final PasswordEncoder passwordEncoder;
    private final AccountRepository accountRepository;
    private final MailService mailService;
    private final JWTUtil jwtUtil;   // bestehende Klasse, neue Methode für Reg-Token

    /** Schritt 1: Registrierungstoken erzeugen und per Mail versenden. */
    public void starteRegistrierung(RegisterDto dto) {
        if (accountRepository.existsByUsername(dto.username())) {
            throw new UsernameAlreadyExistsException(dto.username());
        }

        String passwordHash = passwordEncoder.encode(dto.password());
        String regToken = jwtUtil.generateRegistrationToken(
                dto.username(), dto.email(), passwordHash, TOKEN_VALIDITY_HOURS);

        mailService.sendRegistrationVerification(dto.email(), regToken);
    }

    /** Schritt 2: Token validieren, Daten extrahieren, Account anlegen. */
    public PendingRegistration bestaetigen(String token) {
        // JWTUtil wirft JwtException wenn Signatur oder exp ungültig
        Claims claims = jwtUtil.parseRegistrationToken(token);

        String username  = claims.getSubject();
        String email     = claims.get("email",  String.class);
        String pwHash    = claims.get("pwHash", String.class);

        if (accountRepository.existsByUsername(username)) {
            throw new UsernameAlreadyExistsException(username);   // bereits aktiviert
        }

        return new PendingRegistration(username, email, pwHash);
    }
}
```

## Ablauf

```

  |   { username, email, pw }     |                           |                         |
  |                               |-- starteRegistrierung()-> |                         |
  |                               |                           |-- existsByUsername?      |
  |                               |                           |-- passwordEncoder.encode |
```

## Hinweis zur Persistenz

Da keine serverseitige Map mehr benötigt wird, überleben Pending-Registrierungen Neustarts der Anwendung automatisch. Der einzige Zustand, den der Server kennen muss, ist der Signaturschlüssel (bereits in `JWTUtil`).
